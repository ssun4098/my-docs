# DNS(Domain Name System)
![DNS](/etc/dns.gif)
- IP주소를 일일이 외울 수 없기 때문에 쉽게 기억할 수 있는 도메인 주소 체게
- DNS는 할당된 도메인 영역에 대한 정보를 가지고 있는 서버로 주로 도메인을 IP주소로 변환하는 역할을 한다.
<br/>

# DNS Record
- 인터넷에서 사용자가 도메인과 연결될 때 네임서버에 질의하여 설정된 값을 찾아 통신하는데 이 때 설정된 값을 레코드라고 한다.
- 도메인 네임에 대한 기타 정보를 구성하고 제어할 수 있는 DNS 데이터베이스에 저장된 특정 리소스 레코드(DNS 레코드에 설정하여 도메인에서 사용할 메일 서버 유형이나 IP 연결 설정)

## Record 종류
- A 레코드: IP 주소와 도메인 주소를 매핑할 때 사용하는 레코드로써 하나의 도메인에 여러 개의 IP 주소를 등록할 수 있다.
- CNAME: 도메인 별명 레코드라고 부르며 Alias라고 하여 기존 도메인에 별명을 붙인 레코드
- MX: 메일 서버 레코드, 해당 도메인과 연동되어 있는메일서버를 확인하는데 사용하는 레코드
- NS: 네임서버 레코드로 도메인에 대한 네임서버의 권한을 가지고 있는지 알려주는 레코드

## CNAME vs A 레코드
||장점|단점|
|---|---|---|
|A 레코드|도메인이 바뀌어도 IP는 그대로 이므로 유지가 된다|서버 이전등의 문제로 IP가 변동될시에 일일히 변경해야 한다.|
|CNAME|서버 이전등의 문제로 IP가 변동될시에 변경하지 않아도 된다.|메인이 바뀌면 변경해야 한다. 그리고 여러번 요청이 될경우 성능 저하가 일어날 수가 있다.|
