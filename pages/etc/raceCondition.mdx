# Race Condition
여러 프로세스나 스레드가 동시에 실행될 때 그들이 공유 자원에 접근하거나 수정하는 순서에 따라 결과값이 달라지는 현상

# 문제점
- 예측 불가능한 결과: 같은 입력값이여도 매번 실행될 때 마다 결과값이 달라질 수 있다.
- 일관성 문제: 동시에 데이터가 변경이 일어나면 잘못된 데이터가 들어갈 수 있다.
- 디버깅 어려움: 문제가 특정 조건에만 발생할 수 있기 때문에 추적하기 힘들어진다.

# 해결방법
## Mutual Exclusion(상호 배제)
- 하나의 스레드만이 임계 영역에 진입할 수 있도록 하는 동기화 기법
- 뮤텍스를 지정하고 해당 데이터에 Lock을 걸어서 해당 스레드만이 데이터에 접근하도록 지정

### 임계 영역이란?
- 둘 이상의 스레드가 동시에 접근해서 안되는 공유 자원을 접근하는 코드의 일부

## Atomic Operations(원자적 연산)
- 시작되면 중간에 중단되지 않고 완전히 수행되는 연산
- 완료되거나 시작하지 않거나라는 뜻의 모두 아니면 없음(all or nothing)의 특성을 가짐
- 완료되기 전에 다른 스레드에서 접근할 수 없기 때문에 계산 오류가 일어날 일이 없다.

## Thread Safe
- 공유데이터를 수정하는 함수나 객체에 스레드 안전하도록 구현
- 스레드 동기화 기법이나 불변 객체 적용
