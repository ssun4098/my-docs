# String이 불변 객체인 이유는?
- String은 기본 타입이 아닌 참조 타입이라는 것
- Java에서 String은 String constant pool에서 따로 관리 => 새 객체를 생성하는 것이 아닌 이 풀에서 기존 객체를 반환하여 메모리 사용량을 줄이고 성능이 최적화 된다.
- 네트워크 연결이나 파일 경로 같은 중요한 정보를 String으로 표현되는 경우, String이 불변이 아니면 중간에 변경 가능성이 있기 때문에 보안 문제가 줄어든다.
- 여러 스레드에서 접근하더라도 불변이기 때문에 보안 문제가 줄어든다.
- hashCode를 캐싱할 수 있다.

# hashCode란?
- Object 클래스에 정의된 메서드로, 객체의 메모리 번지를 이용해서 만든 객체를 식별하는 하나의 정수값
- 두 객체가 equals 메서드를 통해 동일하다고 판단된다면, 이 두 객체의 hashCode 값도 동일해야한다.
