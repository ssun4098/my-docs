# @ModelAttribute
- 메소드와 파라미터에서 사용할 수 있다.
- HTTP Request에서 Url Param이나 Multipart/form-data 형태의 요청을 객체로 바인딩한다.

# Data Bind 기준
- 가장 적절한 생성자를 인스턴스하기 때문에 기본 생성자와 모든 필드의 생성자 둘이 있을 경우 데이터가 하나라도 없으면 기본생성자로 생성되기 때문에 데이터 바인딩이 안되는 경험이 있을 수 있다.
- 생성한 뒤에는 바인딩되지 않는 값들은 Setter를 이용해서 바인딩 하지만 Setter를 이용할 경우 불변성이 깨질 수 있다.
- 결국 적절한 생성자를 이용해서 Binding 하는 것이 Best!
